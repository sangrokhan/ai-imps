# AI Implementation Agent Guide

이 문서는 AI 논문 구현 및 연구 개발 업무를 수행할 때 준수해야 할 가이드라인을 정의합니다.

## 1. 논문 구현 절차 (Paper Implementation Workflow)

주인님(User)으로부터 특정 논문 구현 요청을 받았을 때 아래 단계를 순차적으로 수행합니다.

1.  **브랜치 생성**: 작업 시작 전, 현재 개발 내용을 담을 새로운 브랜치를 생성합니다.
2.  **논문 자료 보관**: 해당 구현 폴더(`implementations/yYYYY_name/`) 하위의 `papers/` 폴더에 논문 PDF 파일을 위치시킵니다.
3.  **설정 파일 생성**: `configs/` 폴더 아래에 해당 구현을 실행하기 위한 YAML 설정 파일을 생성합니다.
4.  **모델 구현**: 논문의 핵심 아키텍처와 로직을 `model.py` 등에 구현합니다.
5.  **테스트 코드 작성 및 검증**:
    *   `tests/` 폴더 내 관련 테스트 파일에 테스트 케이스를 추가합니다.
    *   `pytest`를 실행하여 모든 테스트가 통과(`Passed`)하는지 확인합니다.
    *   환경 변수(`PYTHONPATH=.`)와 가상환경(`venv`)을 반드시 확인합니다.

## 2. 형상 관리 및 커밋 규칙

1.  **사전 확인**: 작업을 시작하기 전, 현재 열려 있는 PR(Pull Request)이 있는지 확인합니다. 만약 있다면 주인님께 알려 머지 여부를 상의합니다.
2.  **Rebase 필수**: 커밋을 생성하거나 PR을 올리기 전, 반드시 최신 `main` 브랜치를 기준으로 `rebase`를 수행하여 이력을 정리합니다.
3.  **Sign-off**: 모든 커밋은 반드시 `-s` 옵션을 사용하여 작성자 서명(Sign-off)을 포함합니다.
4.  **커밋 메시지**: 명확하고 간결하게 작업 내용을 기술합니다.

## 3. 체크리스트 (Summary)

- [ ] 구현 폴더 내 논문 PDF 존재 여부
- [ ] `configs/` 내 전용 설정 파일 존재 여부
- [ ] `pytest` 통과 여부 (신규 및 기존 테스트 전체)
- [ ] 최신 Base 브랜치 기준 Rebase 완료 여부
- [ ] 커밋 메시지 Sign-off 포함 여부
